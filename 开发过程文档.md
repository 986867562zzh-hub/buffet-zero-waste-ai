## 一、项目需求分析

### 1.1 问题背景

大型综合度假村的自助餐厅每天产生大量食物浪费，表现为两个层面：
- **消费者端**：取餐过多吃不完，餐盘剩余食物直接丢弃
- **餐厅端**：备餐过量，自助餐台剩余菜品无法有效二次利用

### 1.2 产品需求

| 产品 | 目标用户 | 核心功能 | 价值主张 |
|------|---------|---------|---------|
| **产品一：剩食评分折扣系统** | 消费者 | 拍照识别剩食 → 生成浪费评分 → 对应折扣 | 经济激励减少消费者浪费 |
| **产品二：剩菜智能搭配系统** | 消费者+餐厅 | 拍照识别剩余菜品 → 按需搭配 → 折扣二次售卖 | 剩菜产生二次价值 |

### 1.3 核心功能要求

**产品一详细需求：**
- 消费者用餐后拍摄餐盘照片上传
- AI自动识别剩余食物种类和剩余量
- 生成0-100分的浪费评分
- 根据评分对应折扣（光盘=85折，少量=9折，中等=95折，严重=原价）
- 考虑不同食物类型的浪费权重（肉类浪费比蔬菜严重）

**产品二详细需求：**
- 酒店工作人员拍摄自助餐台剩余菜品（多张照片覆盖不同餐区）
- AI批量识别所有剩余菜品
- 消费者选择饮食需求（28种分类：减脂、增肌、素食、糖尿病友好等）
- AI根据饮食需求智能搭配套餐
- 以折扣价二次售卖（低于市场价的剩菜套餐）

### 1.4 饮食需求分类设计

经过深度思考，设计了**7大维度、28种需求类型**：

1. **健康目标**：减脂瘦身、增肌塑形、体重维持、排毒清体
2. **饮食模式**：低碳水/生酮、地中海饮食、DASH饮食、间歇性断食配合、东方养生
3. **特殊医学需求**：糖尿病友好(低GI)、低嘌呤(痛风)、低FODMAP、低钠、无麸质、无乳制品
4. **特定人群**：儿童营养餐、银发族易咀嚼、孕期营养、运动恢复餐
5. **素食/信仰**：蛋奶素食、纯素食、鱼素、清真饮食
6. **口味场景**：快捷工作餐、轻食沙拉系、暖胃家常味、异国风味、高热量满足
7. **营养元素关注**：高蛋白、高纤维、高铁补血、高钙护骨、抗氧化/美容、Omega-3、低糖控糖

## 二、技术方案设计

### 2.1 技术选型

| 层级 | 技术选择 | 理由 |
|------|---------|------|
| **AI视觉** | Anthropic Claude Vision API (真实模式) / PIL图像分析 (演示模式) | 双模式设计：有API Key时用真实AI，否则用实际图像分析 |
| **后端** | Python Flask 3.x | 轻量级、快速开发、适合原型演示 |
| **前端** | Bootstrap 5 + 原生JavaScript | 响应式设计、移动端适配、无需构建工具 |
| **图像处理** | Pillow (PIL) | 智能Demo模式下的颜色分析、边缘检测 |
| **数据存储** | Flask Session (内存) | 原型阶段，无需数据库即可运行 |

### 2.2 系统架构

```
┌──────────────────────────────────────────────────────┐

│  Routes: /product1/analyze, /product2/identify/match  │

└──────────────────────────────────────────────────────┘
```

### 2.3 AI引擎双模式设计

**模式一：Real AI Vision (需要API Key)**
- 调用 Anthropic Claude Vision API
- 真实识别照片中的食物种类、剩余量
- 返回结构化的JSON分析结果
- 模型：`claude-sonnet-4-6`

**模式二：Smart Demo (无需API Key)**
- 使用PIL库实际分析上传图像
- 检测颜色分布（RGB→Hue转换）
- 根据颜色特征映射食物类别：
  - 绿色区域(hue 35-85) → 蔬菜/绿叶菜
  - 红棕色区域(hue 0-20) → 肉类/红肉
  - 白色/黄色区域(hue 10-30) → 米饭/面食
  - 橙色区域(hue 15-45) → 橙黄色食物
- 计算餐盘覆盖率估算浪费程度
- 诚实标注为"基于图像实际分析"
- **不是随机生成，而是分析实际图像内容**

### 迭代六：Bug修复 - 饱和度阈值bug（6月11日 01:20-01:30）

**问题**：Smart Demo模式识别出0道菜品

**根因**：饱和度计算公式 `sat = (max_c - min_c) / max_c` 返回0-1范围的值，但 `COLOR_FOOD_MAP` 中的 `s_min` 阈值是0-100范围的整数（如40、50、60）。0-1的值永远不大于20-60的阈值。

**修复**：在两处饱和度比较代码中，将 `saturation >= s_min` 改为 `saturation * 100 >= s_min`

**影响**：修复前识别0道菜，修复后正确识别多道菜品

### 迭代七：Bug修复 - 文件扩展名处理（6月11日 01:15）

**问题**：API接口保存文件时统一使用 `.jpg` 扩展名，但实际上传的可能是PNG格式，导致PIL无法识别

**修复**：创建 `_save_upload()` 函数，保留原始文件扩展名，并增加文件头字节检测作为fallback

### 5.3 评分算法

```
加权浪费评分 = (1 - Σ(每道菜剩余% × 类别权重) / Σ(类别权重) / 100) × 100

类别权重:
  肉类 1.5  |  海鲜 1.5  |  甜点 1.2  |  主食 1.0
  蔬菜 0.8  |  汤品 0.7  |  水果 0.6  |  饮品 0.5

折扣映射:
  95-100分 → 5星 光盘英雄 → 85折
  85-94分  → 4星 少量剩余 → 9折
  70-84分  → 3星 中等剩余 → 95折
  50-69分  → 2星 较多剩余 → 原价
  30-49分  → 1星 严重浪费 → 原价
  0-29分   → 0星 极端浪费 → 原价+警告
```

### 6.1 SmartImageAnalyzer - 智能图像分析

```python
class SmartImageAnalyzer:
    """基于PIL的实际图像分析器 - 分析真实照片内容"""

    # 颜色→食物类别映射表 (HSV范围)
    COLOR_FOOD_MAP = [
        # (Hue范围, 最低饱和度, 最低亮度), 食物类别, 典型菜品
        ({"h_min":35, "h_max":85, "s_min":40, "v_min":30}, "蔬菜/绿叶菜", 
         ["炒青菜", "西兰花", "菠菜", "生菜沙拉"]),
        ({"h_min":0,  "h_max":20, "s_min":50, "v_min":40}, "肉类/红肉", 
         ["红烧肉", "牛排", "叉烧", "烤鸡腿"]),
        # ... 共10条映射规则
    ]

    @staticmethod
    def analyze_plate_waste(image_path):
        """实际分析餐盘照片"""
        img = Image.open(image_path).convert('RGB')
        pixels = list(img.getdata())
        
        white_count = 0   # 白色盘子背景
        dark_count = 0    # 深色酱汁
        color_bins = Counter()  # 颜色分类计数
        
        for r, g, b in pixels:
            brightness = (r + g + b) / 3
            max_c = max(r, g, b)
            min_c = min(r, g, b)
            saturation = (max_c - min_c) / max_c if max_c > 0 else 0
            
            if brightness > 200 and saturation < 0.15:
                white_count += 1  # 白色餐盘
            elif brightness < 50:
                dark_count += 1   # 深色酱汁
            else:
                hue = rgb_to_hue(r, g, b)
                # 匹配颜色→食物类别
                for rule in COLOR_FOOD_MAP:
                    if matches(rule, hue, saturation * 100, brightness):
                        color_bins[rule.food_type] += 1
                        break
```

### 6.2 RealAIVision - 真实AI集成

```python
class RealAIVision:
    """真实AI视觉识别 - 调用Anthropic Claude Vision API"""
    
    def analyze_plate_waste(self, image_path):
        """调用Claude Vision分析餐盘浪费"""
        with open(image_path, 'rb') as f:
            img_data = base64.b64encode(f.read()).decode('utf-8')
        
        response = self.client.messages.create(
            model="claude-sonnet-4-6",
            max_tokens=2000,
            temperature=0.1,
            messages=[{
                "role": "user",
                "content": [
                    {"type": "image", "source": {
                        "type": "base64", 
                        "media_type": "image/jpeg", 
                        "data": img_data
                    }},
                    {"type": "text", "text": PROMPT_WASTE_ANALYSIS}
                ]
            }]
        )
        return json.loads(response.content[0].text)
```

### 6.3 AIEngine - 统一入口

```python
class AIEngine:
    """自动选择使用真实AI或Smart Demo"""
    
    def __init__(self):
        api_key = os.environ.get('ANTHROPIC_API_KEY', '').strip()
        if api_key:
            self.real_ai = RealAIVision()
            self.mode = 'real_ai'
        else:
            self.mode = 'smart_demo'
        
    def analyze_plate_waste(self, image_path):
        if self.mode == 'real_ai':
            result = self.real_ai.analyze_plate_waste(image_path)
            if result:
                return result
        return SmartImageAnalyzer.analyze_plate_waste(image_path)
```
